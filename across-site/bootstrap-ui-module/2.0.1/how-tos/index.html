<div class="sect1"> <h2 id="_customizing_a_datetime_picker"><a class="anchor" href="#_customizing_a_datetime_picker"></a>Customizing a datetime picker</h2> <div class="sectionbody"> <div class="paragraph"> <p>Every <code>DateTimeFormElement</code> will be turned into a datetime picker as seen in the section <a href="#DateTimeFormElementBuilder">[DateTimeFormElementBuilder]</a></p> </div> <div class="paragraph"> <p>Sometimes you want to attach a custom javascript event to this datetime picker.</p> </div> <div class="paragraph"> <p>In the following example we will attach an <code>on change</code> event, to refresh a portion of the screen with an AJAX call.</p> </div> <div class="listingblock"> <div class="content"> <pre class="highlightjs highlight"><code class="language-javascript hljs" data-lang="javascript">BootstrapUiModule.registerInitializer( function( node ) { $( '[data-bootstrapui-datetimepicker]', node ).first().datetimepicker().on( 'dp.change', function( event ) { $.get( Across.AdminWebModule.rootPath + "/getdata?_partial=::summary&selectedDate=" + event.date.format( "YYYY-MM-DD" ), function( data ) { $( '#form-right-column' ).replaceWith( data ); } ); } ); }, true );</code></pre> </div> </div> <div class="paragraph"> <p>The <code>BootstrapUiModule.registerInitializer()</code> ensures that your function is registered after that bootstrap-ui has loaded.</p> </div> <div class="paragraph"> <p>After this is done you can register on the <code>dp.change</code> to do the AJAX call. See <a href="http://eonasdan.github.io/bootstrap-datetimepicker/Events/" class="bare">http://eonasdan.github.io/bootstrap-datetimepicker/Events/</a>; for more events supported by this datetime picker.</p> </div> <div class="paragraph"> <p>The AJAX call will render a partial and replace the content.</p> </div> <div class="admonitionblock important"> <table> <tr> <td class="icon"> <i class="fa icon-important" title="Important"></i> </td> <td class="content"> If your callback function is not executed inside <code>BootstrapUiModule.registerInitializer()</code>, then this might be due to the fact that your code is executing before that <code>bootstrapui-formelements.js</code> is loaded. Sometimes Across fails to include this package in time. If this is the case, you can add this WebResource by adding it to your Controller as shown in the example below. </td> </tr> </table> </div> <div class="listingblock"> <div class="content"> <pre class="highlightjs highlight"><code class="language-java hljs" data-lang="java"> @ModelAttribute public void registerJavascript( WebResourceRegistry registry ) { registry.addPackage( BootstrapUiFormElementsWebResources.NAME ); registry.addWithKey( WebResource.JAVASCRIPT_PAGE_END, "data-js", "/static/omega/js/data.js", WebResource.VIEWS ); }</code></pre> </div> </div> <div class="admonitionblock tip"> <table> <tr> <td class="icon"> <i class="fa icon-tip" title="Tip"></i> </td> <td class="content"> If you encounter an <code>undefined</code> error when calling <code>datetimepicker()</code>, this might be due to including jQuery twice in your DOM. </td> </tr> </table> </div> </div> </div>
